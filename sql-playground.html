What this taught me

A couple of lessons baked into this lab

A denormalized column and a history table can both be "right"

The SaaS dataset keeps a user's current plan on the users table for easy filtering, and a full plan-change history in a separate table — the same trade-off real schemas make constantly.

Window functions replace a lot of awkward subqueries

"Top result per group" used to mean a correlated subquery or a self-join. RANK() OVER (PARTITION BY...) does it in one clean pass — worth reaching for before anything more contorted.
